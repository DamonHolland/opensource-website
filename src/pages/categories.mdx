# New Relic Open Source Categories

This document provides the details on the New Relic Open Source Category requirements, including the markdown snippet that should be pasted into the top of a given `README.md` file.

**Global New Relic Open Source requirements**

- Security and licensing scanned by New Relic tooling
- Implements the New Relic Contributor Licensing Agreement

## Category Headers

Every public repository in the `newrelic` and `newrelic-experimental` GitHub organizations must leverage one of the following categories.

- Community Project
- Example Code
- New Relic One Catalog Project
- New Relic Experimental
- Product Delivered in Open Source
- Archived

See the category descriptions, header images, and code snippets below.

## [Community Project](#community-project)

<a href="/categories/#community-project"><img src={Community_Project} alt="Community project header" style={{width: '95%'}} /></a>

### Category Description

This code is developed in the open with input from the community through issues and PRs. There is an active maintainer team within New Relic, troubleshooting support in the [New Relic Explorers Hub](https://discuss.newrelic.com), and the documentation is available in the project repository.

### Category Requirements

- Complies with New Relic's legal and security requirements for open source software
- Contains the header below at the top of the repository's README file
- has active maintainer / maintainers, including at least one Relic
- troubleshooting support via the [New Relic Explorers Hub](https://discuss.newrelic.com)
- Issues and PRâ€™s managed in GitHub
- Documentation reviewed by the New Relic documentation team
- Linted code
- An automated release pipeline

## [New Relic One Catalog Project](#new-relic-one-catalog-project)

<a href="/categories/#new-relic-one-catalog-project"><img src={New_Relic_One_Catalog_Project} alt="New Relic One Catalog project header" style={{width: '95%'}} /></a>

### Category Description

This code is a part of the New Relic One Catalog. It meets the criteria for a `Community Project`; but it also contains UI workflows for configuration. Most `Catalog` projects seek to maintain a public roadmap, often expressed in a GitHub Project board and Issues within the repository.

### Category Requirements

All the requirements of a `Community Project` as well as
- The empty state application workflow that guides users through the setup of configuration data that is stored in NerdStorage
- Architectural review (including UX) by New Relic
- (Optional) maintains a public roadmap (recommended via a GitHub project in the repo).

## [New Relic Experimental](#new-relic-experimental)

<a href="/categories/#new-relic-experimental"><img src={Experimental} alt="New Relic Experimental project header" style={{width: '95%'}} /></a>

### Category Description

This code solves an interesting problem but does not yet have an active maintainer(s). The project is being developed in the open for the purpose of feedback on a new feature or function.

Projects in New Relic Experimental may *graduate* into the https://github.com/newrelic organization under one of the other categories or be archived.

### Category Requirements

- Complies with New Relic's legal and security requirements for open source software
- Contains the header below at the top of the repository's README file
- (Usually) hosted in the [New Relic Experimental](https://github.com/newrelic-experimental) org
- (Optional) Issues at the project owner's discertion
